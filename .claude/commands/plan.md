Задача: $ARGUMENTS

Не пиши код. Сначала: 1) план по шагам с указанием затрагиваемых файлов; 2) раскритикуй собственный план — что может пойти не так, какие риски, есть ли решение проще на уровень выше; 3) итоговый план с учётом критики и критерием приёмки.

Если в задаче есть ключевое решение, дорогое к переделке (архитектура / схема данных / UX / выбор подхода) — перед итоговым планом пройди компактный **Second Thought Gate**:
- **Decision:** решение одной строкой.
- **2 альтернативы:** содержательно иные, не косметика.
- **Слабейшее допущение:** на чём всё держится / что сломает.
- **Keep / Refine / Replace:** выбор с обоснованием.
Не защищай первую мысль — задача её победить. Не можешь убедительно отстоять → пересобери.

После выбора решение = baseline (**anti-thrash**): не переписывай рабочее без чёткого выигрыша, не ради «по-другому». Творчество — на этапе поиска, стабильность — после выбора. Гейт НЕ нужен для тривиального (багфиксы, копи, мелкий CSS).

Жди моего "ок" перед исполнением. После "ок" — исполняй автономно до конца, не спрашивая "что дальше".
