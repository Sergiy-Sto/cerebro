Прогони аудит безопасности проекта перед релизом. Заказчик не технарь — твоя проверка единственная, халтура недопустима. Проверь по списку, по каждому пункту дай вердикт ✅/⚠️/🔴 с пояснением человеческим языком:

1. Секреты: API-ключи, пароли, токены НЕ зашиты в код и НЕ попадают в клиентский бандл / git-историю (.env в .gitignore).
2. API-ключи защищены серверной прослойкой: клиент не вызывает платные API (Anthropic и т.п.) напрямую со своим ключом.
3. Лимиты: есть rate limiting / ограничение расходов там, где запрос пользователя порождает платный вызов.
4. Ввод пользователя валидируется; SQL — только параметризованные запросы; вывод экранируется (XSS).
5. Сообщения об ошибках не светят внутренности (стектрейсы, пути, ключи) наружу.
6. Зависимости: npm audit / аналог — критических уязвимостей нет.
