## 📏 Правила (нарушение любого = стоп и фикс причины)

1. **Инструменты прежде слов.** Перед "не могу" и перед любой просьбой к Заказчику — проверь: есть ли инструмент/MCP/скрипт, который решает это сам? "Дай скрин / проверь у себя" — последний resort. Инвентарь инструментов: см. ниже.
2. **Визуальная правка = скриншот.** CSS/layout/компонент → Chrome MCP скрин ДО и ПОСЛЕ (или сравнение с мокапом из `Мокапы/`). Затронут responsive → оба viewport'а (desktop = Chrome MCP, падает/timeout → СРАЗУ Playwright, не «гляну через DOM» · mobile = Playwright MCP `browser_resize`→`browser_take_screenshot`, не альтернатива а ОБЯЗАТЕЛЬНО). `npm run build` НЕ заменяет глаза. Сказать "готово" без скриншота запрещено (Stop-hook; упавший скрин не считается). На завершении этапа про моб-вид напомнит хук — сними или ответь «моб не нужен».
3. **Функциональная правка = проверка.** `npm run build` + curl/логи + review своего диффа. Минимум build.
4. **Уверенность явная.** "Проверил — работает" ≠ "думаю, сработает". Не смешивать.
5. **2 неудачные попытки → шаг назад.** Переформулируй задачу, ищи решение уровнем выше, не долби в точку.
6. **Инициируй сам.** Видишь лучший инструмент, баг, неэффективный процесс — скажи, не жди вопроса.
7. **Не прерывай Заказчика без нужды.** Спрашивать можно только: продуктовые решения, физически недоступное (реальный телефон, перезагрузка железа). Всё остальное — делай.
8. **Очередь не ждёт.** Закрыл задачу → отчёт ✅ → сразу бери верхнюю [ ] из TODO.md, очередность не переспрашивать (контролирует Stop-hook). Запушил / задал вопрос → бери следующую независимую задачу (не пересекается по файлам, scope ясен). Объяви одной строкой. Крупное/рискованное → отдельная ветка. НЕ параллелить: активный дебаг, визуальные итерации, Заказчик сказал «подожди». Фидбек Заказчика всегда приоритетнее фоновой работы.
9. **Деплой без разрешений.** Пакет готов → проверил → commit → push → дождался деплоя → проверил на проде. Разрешение на push в master не нужно.
10. **Фиксируй WHY по горячим следам.** Закрыл задачу → сразу 1–3 строки в журнал: что и почему так (контролирует Stop-hook). Не копи записи на конец сессии — сессия может умереть внезапно, WHY выветривается.
11. **Сомневаешься в свежести данных → ищи сразу.** Запрещено: отвечать устаревшим и паттерн "моя инфа до X, но вот…". Caveat про cutoff уместен только ПОСЛЕ неудачного поиска.
12. **Мокап перед нетривиальным UX.** Мелкое → сразу в код. Варианты/нетривиальное → мокап. Большое → ветка.
13. **Крупная задача → сначала план.** Plan mode: план → сам раскритикуй его ("что может пойти не так?") → согласование с Заказчиком → длинное автономное исполнение без промежуточных "что дальше?".
14. **Исследование → субагент.** "Разобраться как устроено X" / прочитать 10+ файлов → отправь субагента, в основной контекст возвращай только выжимку. Рутину (поиск/чтение/проверки) гоняй на haiku — дешевле, качества хватает. Субагенту давай чистые инструменты (Glob/Read/diff, простые команды); опасное у него тоже ловит `guard-subagent`.
15. **Permission-промпт вылез → стоп, не жми Allow слепо.** Найди причину. Обычная причина — Bypass не активен в сессии: включи (`Ctrl+Shift+M` → `5`). Под Bypass обычные команды (вкл. `&&`/`;`) НЕ спрашивают — дробить не нужно. Предпочитай `git -C` / `npm --prefix` / абс.пути вместо `cd "путь" && …` (чище, не ради промпта). Очень длинный многострочник — лучше в `scripts/` ради читаемости (не запрет). Правки `.claude/` спрашивают ВСЕГДА (by design даже в bypass) — делай редко, одним заходом.
16. **Одна задача = одна сессия.** Задача закрыта → `/journal` с блоком «📍 ПЕРЕДАЧА» (мини-ТЗ следующей сессии: где остановились, что первым, грабли) → предложи новую сессию. Контекст/длину сессии НЕ отслеживай и НЕ поднимай — это полностью забота Заказчика. Никаких «похоже, контекст высокий», пауз или вопросов «не пора ли новую сессию» из-за объёма работы: работай не останавливаясь, крупное начинай смело.
17. **Крупное решение → Second Thought Gate.** Решение, дорогое к переделке (архитектура / UX / схема данных / выбор подхода) → не хватай первое рабочее: в `/plan` пройди гейт (Decision · 2 альтернативы · слабейшее допущение · Keep/Refine/Replace), цель — победить свою первую мысль. После выбора решение = baseline (**anti-thrash**): меняй только при чётком выигрыше, не ради «по-другому». Тривиальное (багфиксы, копи, мелкий CSS) — мимо гейта.

## 🧰 Инвентарь инструментов (пополнять при каждом открытии нового)
- Скриншоты, клики, консоль, network (живой браузер, desktop): Chrome MCP. Скрин падает/timeout → **Playwright** (`browser_navigate`+`browser_take_screenshot`) как fallback
- Mobile viewport / эмуляция: **Playwright MCP** (`browser_resize` → `browser_take_screenshot`) — реальный reflow. Chrome MCP resize НЕ работает (остаётся десктоп-фрейм, вешает рендер). Нет коннектора → провижининг в «ХУКИ И СКРИПТЫ - АРХИТЕКТУРА.md»
- CI/CD-диагностика: `gh run list`, `gh run view <id> --log-failed` (не WebFetch на Actions UI)
- Долгие watch-операции: скрипт в `scripts/` + фоновый Bash
- Параллельные задачи: subagent / worktree-агент

**Мета-правило инвентаря.** Список заведомо неполон. Перечисленное не сработало → прежде чем сказать "не могу": 1) другой MCP/npm-пакет/скрипт? 2) веб-поиск "how to X" / "X MCP"? 3) обходной путь? Слово "невозможно" разрешено только после этого цикла — с перечнем того, что попробовал. Нашёл рабочее → впиши строку сюда.

## 📁 Файлы: что где
- `ЖУРНАЛ.html` — живое состояние. Секции: «📍 ПЕРЕДАЧА» (перезаписываемое мини-ТЗ для следующей сессии), «Как вести журнал», «Текущее состояние фич», «История решений» (root cause багов + WHY решений), «Ключевые файлы», «Модели и промпты». Обновляет Claude в конце сессии / после значимого события.
- Разделение задач (без дублей): **открытые/текущие задачи живут ТОЛЬКО в `TODO.md`**; закрыл задачу → вычеркни из TODO и запиши в `ЖУРНАЛ.html` → «История решений» (что+WHY). В Журнале открытых задач нет.
- Мнемоника: ЧТО делать сейчас → TODO. ЧТО уже сделано и ПОЧЕМУ так → ЖУРНАЛ. КАК работать → CLAUDE.md. КАКОЙ проект (видение/архитектура) → ПРОЕКТ.md. КАК устроен кит (хуки/команды/скрипты) → ХУКИ И СКРИПТЫ - АРХИТЕКТУРА.md.
- `ПРОЕКТ.md` — видение, архитектура, особенности. Только неизменное «как устроено», без журнала и задач.
- `TODO.md` — канбан: 🟢 В РАБОТЕ / 🟧 ОЖИДАЕТ РЕШЕНИЯ ЗАКАЗЧИКА / 🟣 ПОЗЖЕ-ОТЛОЖЕНО / 🧊 ИДЕИ. Очередь (В РАБОТЕ, её гонит todo-drain) — с критерием приёмки («готово = …», топливо Правила 8); Идеи — без «готово=». Куда отнести (сделать сейчас / в TODO / в Идеи) неочевидно → спроси коротко с вариантами; очевидное или по слову Заказчика — без вопроса. Большое → /plan.
- `CLAUDE.md` — только неизменное (правила, стек). Задачи здесь НЕ дублировать.
