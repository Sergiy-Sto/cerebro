История решений

2026-05-25 · Module 01 deep decomposition
Reality Mapping разбит на 9 под-модулей в иерархии Модуль → Stage → Карточки
Решение: Старый "Entity Mapping" (один stage с 16 измерениями в одной куче) переразложен на 9 отдельных под-модулей: 1.1 Observation Scan, 1.2 Search Scan (web search), 1.3 Fundamental Dimensions, 1.4 Obligatory vs Accidental, 1.5 Jobs-to-be-done, 1.6 Users/Actors, 1.7 Failure Modes, 1.8 Substitutes, 1.9 Boundary Cases. Каждый под-модуль = отдельный stage + отдельный API-вызов + свои карточки. В левой колонке теперь иерархия: модули (4 шт) с раскрытием под-модулей.
Почему: Один большой промпт на 8 концепций = модель смешивает блоки и срезает. Разделение даёт точность, контроль (можно перегенерить один блок), прогресс (стрим по каждому). Также Obligatory vs Accidental вынесен в отдельный stage — он критический мост к Module 02 Feature Challenge, в общей куче терялся.
2026-05-25 · Search Scan auto-search
Search Scan — реальный web search через OpenAI Responses API
Решение: Под-модуль 1.2 Search Scan использует встроенный web_search tool через Responses API (новый endpoint /v1/responses). Модель сама делает до 12 поисковых запросов агентски, синтезирует найденное в карточки с источниками. Это light research mode — snippets only, без перехода на сайты. Стоимость ~$0.50-0.70 за Search Scan.
Confidence маркер: search_snippet_supported (НЕ evidence_supported) — сниппет это слабое evidence. Сильное потребует deep research (отложено как кнопка "🔬 Углубиться" в Шаге 3).
Прогресс: стрим — пользователь видит текущий запрос, счётчик 1/12, фазу синтеза. Источники прикрепляются к карточке в Card.sources[] и отображаются как badge "🌐 N".
Архитектура поиска описана детально в Проект.md секция 3.1.
2026-05-25 · model selector обновлён
o-series модели удалены из селектора — retired в феврале 2026
Решение: Убраны o3 и o4-mini из селектора моделей. По данным OpenAI (проверено WebSearch) все o-series ретайрнуты, единственная активная семья — GPT-5.x. Добавлены актуальные: gpt-5.5 Instant (дефолт), gpt-5.4 Thinking, gpt-5.4 Pro, gpt-5.4, gpt-5.4-mini. gpt-4.1 оставлен с пометкой "legacy".
2026-05-24 · search button + fixes
Search Plan убран как отдельный этап — кнопка "Найти в интернете" на Observation Scan
Решение: Этап Search Plan удалён из видимого списка (StageId оставлен в типах для legacy-данных). Вместо него на этапе Observation Scan появляется кнопка "🔍 Найти в интернете" — клик открывает модалку, AI генерирует 6-8 готовых Google-запросов на английском, каждый — кликабельная ссылка открывающая Google в новой вкладке. Кнопка "Открыть все" с staggered timing чтобы браузер не блокировал.
Почему: Отдельный визуальный этап Search Plan создавал лишний клик и нагромождение в левой колонке. Теперь flow проще: Observation Scan → жмёшь "Найти" → смотришь Google → копируешь интересное в Search Notes → Reality Map Summary.
2026-05-24 · hotfix
gpt-5.x не поддерживает custom temperature — добавлен helper
Решение: Введены helpers supportsCustomTemperature(model) и isReasoningModel(model). Для gpt-5.x и o-series temperature не передаётся (только default = 1). Для o-series ещё используется max_completion_tokens.
Почему: При попытке генерации с gpt-5.5 OpenAI возвращал ошибку Unsupported value: 'temperature' does not support 0.85 with this model. Раньше проверка была только model.startsWith('o'), но gpt-5.x тоже не принимает custom temperature.
2026-05-24 · Reality Mapping arch v1
Reality Mapping модуль вместо плоского Definition Deconstruction
Решение: Старый этап "Деконструкция определения" разбит на 4 новых этапа: Observation Scan → Search Plan → Search Notes (ручной ввод) → Reality Map Summary. Далее идёт Entity Mapping (16-измерительный онтологический разбор) и Feature Challenge (бывший Invert, переосмыслен через серию трансформаций).
Почему: Старый Definition выдавал реферат школьника — поверхностный, без связей. Без глубокого фундамента вся цепочка следующих этапов плыла. Reality Mapping заставляет модель сначала собрать карту реальности (что РЕАЛЬНО существует), а не сразу генерировать абстракции. Entity Mapping добавляет ключевое измерение OBLIGATORY vs ACCIDENTAL — точку входа для Feature Challenge.
2026-05-24 · Reality Mapping arch v1
Linked cards — данные сохраняются, UI визуализации откладываем
Решение: Добавлено поле derivedFromIds: string[] в Card. Модель в каждом этапе указывает ID карточек из контекста, на основе которых построена новая карточка. Данные сохраняются и используются в промптах следующих этапов как структурированный контекст. В UI пока не показываем.
Почему: Главное — модель работает связно: каждая инверсия атакует конкретный признак, каждая гипотеза вырастает из конкретной opportunity. Это убирает "генерацию в вакууме". Полноценный UI графа (визуализация связей, навигация по графу, обратные связи) — это серьёзная работа на день-два, при этом методологически данные уже накапливаются и могут быть визуализированы позже.
2026-05-24
Контекст между этапами — ВСЕ карточки, ★ только для приоритизации
Решение: В каждый следующий этап передаются ВСЕ карточки предыдущих этапов как контекст. Раньше передавались только отмеченные ★ — но при auto-all никто не отмечает, и цепочка генерировалась практически в вакууме. Теперь ★ просто помечается в тексте промпта как "особо важная", модель учитывает приоритет, но не игнорирует остальные.
Почему: Главный фейл предыдущей версии — auto-all генерировал каждый этап без контекста, потому что фильтр был status === 'interesting'. Это вырезало всё в свежем проекте.

2026-05-25 · cost log
💰 Cost log реализован — детальный учёт каждого API-вызова
Решение: Каждый вызов OpenAI API (Chat Completions / Responses) логируется в localStorage с полями: project, stage, model, input/output tokens, web search calls, оценочная стоимость. В хедере appliances badge "💰 $X.XX" — клик открывает модалку с breakdown по стейджам / моделям / проектам, фильтр "Сегодня / Всё время".
Зачем: Без cost log оптимизация вслепую. Через день видно "не приложение дорогое, а Hypothesis + Critic + Search жрут 70%" — целенаправленно режем именно их, остальное не трогаем.
Цены: в utils/pricing.ts — таблица моделей gpt-5.x и web search. Обновлять при изменениях OpenAI.
Badge цвета: серый (<$1/день) / янтарь ($1-5) / красный (>$5).
приоритет: ближайший · 2026-05-25
💰 Оптимизация стоимости API — главный рычаг: per-stage selector модели
Что: Один полный прогон проекта = ~$2.30 (17 обычных стейджей × $0.10 + Search Scan $0.60). 2-3 прогона в день = ~$6. При активной разработке это много.
Решение (по приоритету эффекта):
  1. Per-stage selector модели — gpt-5.4-mini в 10× дешевле gpt-5.5 Instant. Простые блоки (Observation, JTBD, Users, Substitutes, Boundary Cases) → mini хватит. Сложные (Feature Challenge, Hypothesis, Critic, Shortlist, Validation, Search Scan синтез) → 5.5. Экономия 50-60%.
  2. Фильтр контекста для late-stage — в Hypothesis Generation сейчас передаётся 15-20K токенов. Можно: только ★ marked карточки полностью + title всех остальных. Экономия 30% на late-stage. Минус: методология чуть слабее если пользователь забыл ★ отметить.
  3. Prompt caching через OpenAI — автоматический кэш для стабильных частей промпта. Нужно структурировать промпты: стабильное (правила формата) в начало, уникальное (контекст) в конец. Экономия 20-30%.
  4. Лимит Search Scan: 8 поисков вместо 12 — $0.60 → $0.40 за Search Scan. Качество чуть меньше.
Совокупно: $2.30 → ~$0.70-0.90 за полный прогон. $6/день → ~$2.50/день.
Когда: следующая итерация после теста Module 01 на 2-3 реальных темах. До того — мониторим реальные расходы (если на тестах вылетит больше $10/день — приоритет растёт).
отложено · 2026-05-24 (когда станет продуктом)
🔒 Бэкенд-прокси для OpenAI API — защита промптов и ключа
Что: Сейчас браузер напрямую стучит в OpenAI API. Любой пользователь через F12 → Network видит: полный system prompt, user prompt (вся методология), API key в Authorization header. Для MVP-теста это норм (юзер один — сам Sergiy, ключ свой), но при продуктизации — катастрофа.
Решение: Добавить простой бэкенд-прокси (например на Cloudflare Workers / Vercel / отдельный Node на Hostinger в /nodejs/ — но не сейчас, не трогать!). Фронт отправляет только {"projectId": "X", "stageId": "observation"}. Бэкенд: собирает промпт, берёт ключ из env, зовёт OpenAI, стримит ответ обратно. Промпты и ключ — невидимы для клиента.
Когда: Перед публичным запуском или когда захочется давать доступ другим людям без раздачи методологии.
