# 🔎 ЛИНЗЫ ПРОВЕРКИ

Набор перспектив для проверки работы. Заказчик активирует фразой → Claude **ОБЪЯВЛЯЕТ**, какие линзы включил, прогоняет и выдаёт **вердикт + конкретные проблемы** (не «ок»). Можно одну, несколько или все.

## Зачем
«Проверь» по умолчанию схлопывается в самое дешёвое — функциональное («build прошёл, элементы на месте, 404 нет»). Дизайнерский и пользовательский слои выпадают, хотя технически видны. Именованная линза + обязательное объявление + вердикт **заставляют включить нужный взгляд** (см. корень в журнале — «27px = норма»).

## Линзы

### 🎨 Дизайнер — «красиво?»
Есть скилл `design:design-critique` → **вызови его** (артефакт: структурно прогоняет критерии, словами не подделать). Нет скилла → оцени сам: воздух · баланс · иерархия · отбивки от соседей/footer · типографика · консистентность с остальным сайтом. Не оправдывай отступ числом — спрашивай «душно или нет», а не «сколько px».
Фразы: «проверь как дизайнер», «дизайн-взгляд», «красиво ли».

### 👤 Пользователь / юзер-тестер — «понятно и удобно?» ⭐ важнейшая
НЕ просто смотришь — **открываешь, кликаешь, читаешь** в живом браузере (Chrome MCP / Playwright). Пройди реальный сценарий как посетитель: открой страницу → пройди путь (нашёл → выбрал → в корзину → оформил) → прочитай реальные тексты → потыкай кнопки/ссылки. Зафиксируй: где затык, что неочевидно, что бесит, не теряется ли путь, что НЕ работает. **Требует реального взаимодействия в браузере**, не чтения кода/DOM и не одного скрина.
Фразы: «глазами пользователя», «как юзер-тестер», «потыкай как пользователь».

### 🔍 Ревьюер — «правильно?»
Код / логика: баги · edge cases · безопасность · регрессии · читаемость диффа. Есть скилл code-review → можно через него.
Фразы: «как ревьюер», «отревьюь код».

### 📱 Моб-юзер — «не ломается на телефоне?»
Playwright `browser_resize` (≤500) → `browser_take_screenshot` → пройди как пользователь на телефоне: тапы · скролл · мелкий текст · перенос вёрстки · не вылезает ли за экран. Реальный reflow, не десктоп-фрейм (Chrome MCP resize не годится).
Фразы: «на телефоне», «моб-вид».

## Активация
- **Одна:** назови линзу — «проверь как дизайнер».
- **Несколько:** перечисли — «как дизайнер и пользователь».
- **Все:** «полное ревью» / «по всем линзам» / «прогони все линзы» → применяй ВСЕ уместные (моб — только если трогали responsive; ревьюер — только если менялся код).

В любом случае Claude **объявляет**, какие линзы включил («🔎 Линзы: Дизайнер + Пользователь»), и даёт **вердикт по каждой** с конкретными проблемами, а не «всё ок». Линза без подходящего инструмента (пользователь/моб без браузера) бесполезна → сначала подними браузер.

⚠️ **Заявил линзу — прогони её по-настоящему.** Хук `lens-claim-check` сверяет: написал «🔎 Линзы: X» → должен быть артефакт (дизайнер = вызов скилла · пользователь = реальные клики · моб = resize≤500), иначе блок финиша. Либо реально прогоняешь, либо НЕ пишешь маркер. Заявление без прогона = отписка «ок», от которой линзы и защищают.
