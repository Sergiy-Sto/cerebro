## 🚨 Что КРИТИЧНО на старте новой сессии
- ⚠️ Хуки активируются только в НОВОЙ сессии (харнесс читает settings при старте). Самотест `rm -rf /несуществующая-папка-тест` → должно блокнуться. Не блокнулось → хуки не подцепились, разобраться.
- 👉 Включи **Bypass permissions** (`Ctrl+Shift+M` → `5`) — кит на него рассчитан, иначе промпты. Desktop игнорит `defaultMode` из settings.json — рычаг только в UI-селекторе, каждую сессию.
- Мобильные скрины = **Playwright MCP** (`browser_resize` 390 → screenshot). Chrome MCP resize не работает. Нет коннектора → провижининг-runbook в арх-доке.
- compound-команды (`&&`/`;`) РАЗРЕШЕНЫ под Bypass; дробить не нужно. Опасное (`rm -rf`, force) всё равно блокируется.
- Коммит-сообщения с подстроками `rm -rf`/`force` → блок guard-bash; пиши message через файл (`.git-commit-msg.tmp` → `git commit -F` → `rm`).
- PowerShell-скрипты: `pwsh` (PS7) + файл, не inline (PS5.1 портит UTF-8 кириллицу).
