1. Суть и цель

Проблема

LLM на запрос «сгенерируй бизнес-идеи» выдаёт банальщину. Если пропустить тему через структурированную цепочку из 10 методов с человеком-куратором на каждом шаге — результат нетривиален. Но человек не может удержать в голове десятки карточек с их связями — нужна внешняя память и навигация.

Что строим сейчас

Методологический MVP, не продукт. Минимум интерфейса, которого хватает чтобы один человек прошёл все 10 этапов без когнитивного утопания. Цель — проверить работает ли методология на 2–3 реальных темах. Потом достраиваем до продуктового MVP.

Намеренные ограничения

Минимализм здесь — не лень, а сознательная дисциплина. Каждая лишняя фича — шум при проверке работает ли методология.

  • НЕТ бэкенда, авторизации, БД — всё в localStorage
  • НЕТ web-поиска, scoring, pipeline-визуализации, notes, linked cards, drag-n-drop, тем
  • НЕТ красивостей — сдержанный функциональный UI
  • ЕСТЬ OpenAI API интеграция — автогенерация карточек через API с jsonschema-валидацией

5. Экраны и UX

Project Picker

Если projects пусты — сразу форма New Project. Иначе список: title, frame, кнопки [Open] [Delete]. Сверху [+ New Project].

New / Edit Project Form

Поля: Title (required), Frame (required), Constraints (список строк), Criteria (список строк). При Save нового — activeStageId = 'definition', selectedCardId = null.

Статусы этапов в левой колонке

  • ✓ — есть ≥1 карточки со статусом interesting
  • ● — этап сейчас активный
  • · — есть карточки, но ни одной interesting
  • ○ — карточек нет

9. Acceptance Criteria

  1. Открываю приложение впервые → вижу форму создания проекта.
  2. Создаю проект → попадаю в Workbench с активным этапом Definition.
  3. Вижу 10 этапов с иконками ○ и активным ● на Definition.
  4. Добавляю карточку вручную → появляется с #1, статус neutral.
  5. Вставляю JSON из 5 карточек → импортируются #2-#6, валидация работает.
  6. Помечаю 2 карточки interesting → у этапа ✓, кнопка Next stage активна.
  7. Перехожу на следующий этап → карточки прошлого этапа доступны при возврате.
  8. Создаю child-карточку с parent из предыдущего этапа → ссылка на родителя кликабельна.
  9. Помечаю карточку discarded → показывается полупрозрачно, не исчезает.
  10. Перезагружаю страницу → всё состояние сохранено.
  11. Экспортирую JSON, импортирую → проект появляется как новый.
  12. Создаю второй проект, переключаюсь → у каждого свой набор карточек.

10. Промпты для этапов (черновик)

Используются отдельно от приложения в ChatGPT/Claude. Пользователь копирует промпт, подставляет переменные, получает JSON-массив, вставляет через Paste JSON.

Общие принципы промптов

  • Один этап — один промпт
  • Строгий выходной формат: только JSON-массив, без преамбул и обёрток
  • Не предлагать решений раньше времени
  • Конкретика важнее количества
  • parentId не указывается моделью — проставляет пользователь вручную в UI
