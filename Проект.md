# Проект — Creative Core Workbench (cerebro)

> Справочный документ проекта. Видение, стек, архитектура.
> Живые задачи и статус фич — в `ЖУРНАЛ.html`.
> Не добавлять сюда текущие задачи.

---

## 1. Суть и цель

### Проблема
LLM на запрос «сгенерируй бизнес-идеи» выдаёт банальщину. Если пропустить тему через структурированную цепочку мыслительных методов с человеком-куратором на каждом шаге — результат нетривиален. Но человек не может удержать в голове десятки карточек со связями — нужна внешняя память и навигация.

### Что строим
**Методологический MVP, не продукт.** Минимум интерфейса, которого хватает чтобы один человек прошёл всю цепочку этапов без когнитивного утопания. Цель — проверить работает ли методология на 2–3 реальных темах.

### Намеренные ограничения
Минимализм здесь — не лень, а сознательная дисциплина. Каждая лишняя фича = шум при проверке методологии.

- Нет бэкенда, авторизации, БД — всё в `localStorage`
- Нет красивостей — сдержанный функциональный UI
- Есть OpenAI API: streaming генерация карточек, JSON-парсинг

## 2. Технический стек

**Используем:**
- Vite + React + TypeScript + Tailwind CSS v4
- `localStorage` (ключ: `creative_core_workbench_v1`)
- `useReducer` для state (без Redux/Zustand)
- OpenAI Chat Completions API (streaming)
- GitHub Actions + FTP Deploy на Hostinger (`cerebro.lexion.pro`)
- `crypto.randomUUID()` для ID

**Не используем:**
- ❌ UI-библиотеки (shadcn / MUI / Chakra)
- ❌ State-management (Redux / Zustand)
- ❌ Бэкенд / БД
- ❌ Анимации, темы оформления

**Поддерживаемые AI модели** (в селекторе хедера):
- `gpt-5.5` (дефолт) · `gpt-5.4` · `gpt-5.4-mini`
- `gpt-4.1`
- `o4-mini` · `o3`

Для gpt-5.x и o-series параметр `temperature` не отправляется (модели его не принимают). Для o-series используется `max_completion_tokens`.

## 3. Методология: 14 этапов

Этапы делятся на 4 модуля:

### Module 1 · Reality Mapping (фундамент)
| # | Этап | Что делает |
|---|---|---|
| 01 | **Observation Scan** | Карта реальности из знаний модели — типы, форматы, проблемы, существующие решения, заменители |
| 02 | **Search Notes** | Ручной ввод найденного в интернете (кнопка "🔍 Найти" на Observation генерирует запросы) |
| 03 | **Reality Map Summary** | Сводка наблюдений + поиска: подтверждённое, неподтверждённое, ключевые направления |

### Module 2 · Deep analysis
| # | Этап | Что делает |
|---|---|---|
| 04 | **Entity Mapping** | Онтологический разбор по 16 измерениям (Purpose, JTBD, Users, Resources, Form, Structure, Dependencies, Environment, Lifecycle, Interfaces, Variants, Constraints, Failure Modes, Substitutes, Boundary Cases, Historical Evolution, Obligatory vs Accidental) |
| 05 | **Feature Challenge** | Атака на "случайные" признаки через трансформации: убрать / инвертировать / автоматизировать / совместить |

### Module 3 · Friction & opportunity
| # | Этап | Что делает |
|---|---|---|
| 06 | **Карта трений** | Friction Map — конкретные болевые точки |
| 07 | **Поиск противоречий** | Contradiction Finder — "X требует Y и одновременно NOT-Y" |
| 08 | **Кросс-доменный перенос** | Cross-field Transfer из других бизнес-секторов |
| 09 | **Дерево возможностей** | Opportunity Tree |

### Module 4 · Synthesis
| # | Этап | Что делает |
|---|---|---|
| 10 | **Генерация гипотез** | Hypothesis Generation (с метриками: Новизна, Сила, Реализация, Проверяемость) |
| 11 | **Критический разбор** | Critic Pass — failure modes для каждой гипотезы |
| 12 | **Шортлист** | Топ-3-5 + dark horse candidates |
| 13 | **План валидации** | Validation tests с метриками |

### Legacy этапы (для старых проектов)
В массиве `STAGES` остаются `definition`, `invert`, `search_plan` с пометкой `[legacy]` — чтобы не падал рендер для проектов созданных до архитектуры Reality Mapping. Генерация на них отключена.

## 4. Модель данных

### Card
```ts
interface Card {
  id: string;                    // crypto.randomUUID()
  number: number;                // #N в рамках этапа
  stageId: StageId;              // 'observation' | 'entity_mapping' | ...
  type: CardType;                // 'observation_item' | 'entity_dimension' | ...
  title: string;
  description: string;
  tags: string[];
  status: 'neutral' | 'interesting' | 'discarded';
  parentId: string | null;       // дочерняя карточка от другой
  createdAt: string;             // ISO
  // AI-генерация:
  metrics?: { novelty, strength, feasibility, testability };
  analysis?: string;             // объяснение метрик
  model?: string;                // 'gpt-5.5' / 'o4-mini' / etc.
  // Reality Mapping arch:
  confidence?: 'assumed' | 'user_provided' | 'observed' | 'evidence_supported';
  notes?: string;                // пользовательские заметки
  derivedFromIds?: string[];     // граф связей (данные сохраняются, UI отложен)
}
```

### Project
```ts
interface Project {
  id, title, frame, constraints[], criteria[];
  cards: Card[];
  activeStageId: StageId;
  selectedCardId: string | null;
  createdAt, updatedAt;
}
```

### localStorage
```
key: "creative_core_workbench_v1"
{ projects: Project[], activeProjectId: string | null }
```

## 5. Экраны и UX

### Project Picker
Пусто → форма New Project. Иначе список с `[Open]` / `[Delete]` + `[+ New Project]`.

### New / Edit Project Form
Поля: Title (required), Frame (required), Constraints[], Criteria[]. При сохранении нового — `activeStageId = 'observation'`, авто-старт генерации если есть API ключ.

### Workbench — 3 колонки

| Колонка | Ширина | Содержимое |
|---|---|---|
| **Stages** | ~200px | 14 этапов + 3 legacy, иконки статусов, счётчик карточек |
| **Cards** | ~360px | Список карточек активного этапа, кнопки Генерировать / Думай ещё / Найти (Observation) / + Добавить |
| **Card Detail** | flex | Детали карточки: title, description, tags, метрики, status, model badge, action-кнопки |

### Header
Слева — название и краткое описание проекта. Справа:
- Git hash (кликабельный → ссылка на коммит)
- Селектор модели AI
- `🚀 Запустить всё` / `🔄 Перегенерировать` (auto-all)
- `⚡ AI готов` / `⚙ Ввести API ключ`
- Экспорт / Импорт JSON
- Проекты (обратно в Picker)

### Статусы этапов в левой колонке
- `✓` — есть ≥1 карточки `interesting`
- `●` — активный этап
- `·` — есть карточки, ни одной interesting
- `○` — карточек нет

### Auto-all (`🚀 Запустить всё`)
Последовательная генерация всех этапов: модель генерирует Observation Scan → Reality Map Summary → Entity Mapping → ... → Validation. Search Notes (ручной ввод) пропускается. Legacy-этапы пропускаются. Каждый этап получает все карточки предыдущих как контекст.

### Search Modal (на Observation Scan)
Кнопка `🔍 Найти в интернете` появляется когда есть карточки. Клик → AI генерирует 6-8 коротких Google-запросов на английском → каждый клик открывает Google в новой вкладке. Кнопка "Открыть все" со staggered timing чтобы браузер не блокировал.

## 6. Промпты (где живут)

Промпты для каждого этапа — в `src/utils/prompts.ts` в коде. Не копируются в ChatGPT отдельно — модель вызывается напрямую через OpenAI API из приложения.

**Структура промпта каждого этапа:**
- Контекст проекта (тема + ограничения + критерии)
- Карточки предыдущих этапов (все, ★-карточки помечены приоритетом)
- Описание задачи этапа
- Правила формата (JSON по строке, на русском)
- Поле `derived_from: ["id1", "id2"]` — связи с карточками предыдущих этапов

Эволюция промптов фиксируется в `prompts.ts`: текущие — наверху, legacy — внизу с пометкой `[LEGACY STAGE]`.

## 9. Анти-скоуп (для текущей итерации)

Это не «когда-нибудь». Явный запрет для текущей фазы — методологического теста.

❌ Не делаем:
- UI-визуализация графа связей `derivedFromIds` (данные сохраняем, визуализацию откладываем)
- Авторизация, шеринг проектов
- Drag-n-drop карточек
- Анимации, темы оформления
- Многоязычность интерфейса
- Undo / redo
- Поиск / фильтр карточек по тексту
- Сортировка карточек (только по `number`)
- Scoring агрегированный (метрики есть на карточке, но без общего рейтинга гипотез)

✅ Делаем (отступления от изначального анти-скоупа):
- **Linked cards** (`derivedFromIds`) — сделано как data-only, без UI. Модель указывает связи в JSON, они хранятся, передаются в промпты следующих этапов.
- **Notes** на карточках — поле есть в типе, UI откладываем
- **Web-поиск** — реализован через кнопку `🔍 Найти в интернете` (открывает Google, ручной ввод результатов в Search Notes)

## 10. Acceptance Criteria для MVP

1. Открываю приложение впервые → форма создания проекта
2. Создаю проект → попадаю в Workbench, активен Observation Scan, авто-старт генерации
3. Кнопка "🚀 Запустить всё" последовательно генерирует все этапы (кроме Search Notes)
4. На Observation Scan есть кнопка "🔍 Найти в интернете" → модалка с Google-запросами
5. Метки `★ Отметить` / `✗ Отклонить` работают, видны на карточке и в списке
6. Создаю child-карточку с parent из предыдущего этапа → ссылка кликабельна
7. Перезагружаю страницу → состояние сохранено
8. Экспорт / импорт JSON работает
9. Auto-all с уже заполненным проектом → подтверждение "очистить и сгенерировать заново"
10. Старые проекты (legacy `definition` стейдж) открываются без крэша
