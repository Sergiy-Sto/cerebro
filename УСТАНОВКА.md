# 🚀 Установка кита в новый проект (~5 минут)

## 1️⃣ Скопировать содержимое архива в корень проекта
Структура после копирования:

```
твой-проект/
├── CLAUDE.md           ← правила работы (заменить; старый сохрани как CLAUDE-old.md)
├── ПРОЕКТ.md           ← шаблон: видение/архитектура (заполнить)
├── ЖУРНАЛ.html         ← шаблон: живое состояние (пустой, со структурой)
├── TODO.md             ← шаблон: очередь задач
├── scripts/            ← самотесты хуков (test-visual-check.sh, test-capitulation.sh)
└── .claude/
    ├── settings.json   ← режим + регистрация хуков
    ├── .gitignore-note.txt
    ├── commands/       ← /deploy /journal /next /plan /rules /seccheck
    └── hooks/          ← guard-bash, guard-files, visual-check, capitulation-check, journal-check, remind-rules
```

## 2️⃣ Подправить под проект
- `guard-files.js`: впиши реальные защищённые пути в список PROTECTED (см. комментарий внутри).
- `visual-check.js`: при нужде добавь свои локальные доки в `docExemptRe` (уже исключены ЖУРНАЛ.html, ПРОЕКТ.md, TODO.md, CLAUDE.md и `.claude/`).
- `remind-rules.js`: интервал напоминания правил — переменная `N` (по умолчанию 15).
- `ПРОЕКТ.md` / `ЖУРНАЛ.html` / `TODO.md` — шаблоны, наполняй по ходу (ПРОЕКТ — на старте первой сессии).

## 3️⃣ Активировать
Открыть **новую** сессию Claude Code в папке проекта (старая настройку не подхватит).

## 4️⃣ Проверить хуки
- 🧨 `rm -rf /несуществующая-папка-тест` → ожидаемо **«ЗАБЛОКИРОВАНО ХУКОМ»**. Выполнилось без блока → хуки не подцепились, разберись почему.
- ✅ `bash scripts/test-visual-check.sh` и `bash scripts/test-capitulation.sh` → все кейсы PASS.

## Что внутри (коротко)
- **Гарды механические:** deny-правила + PreToolUse-хуки блокируют опасное даже в bypassPermissions (force-git, rm -rf корня, .env).
- **Stop-хуки** принуждают дисциплину: `visual-check` (скриншот после визуальной правки), `journal-check` (фиксация по горячим следам), `todo-drain` (брать следующую задачу), `capitulation-check` (искать решение, а не сдаваться).
- **UserPromptSubmit** `remind-rules` периодически впрыскивает правила (борьба с «выцветанием» в длинной сессии).

## ⚠️ Честные ограничения
- Stop-хуки эвристичны: иногда срабатывают лишний раз. Это нормально — лучше, чем ничего.
- Промпты на правки `.claude/` и на «обфусцированный» shell — **встроенная** защита Claude Code, конфигом не отключается (by design). Лечится дисциплиной: shell сложнее одной команды → в `scripts/`.
- Хуки требуют Node.js.
