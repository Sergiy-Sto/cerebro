# 🧭 ХУКИ И СКРИПТЫ — АРХИТЕКТУРА системы работы

> Что делает каждый хук / команда / скрипт и ПОЧЕМУ. Читать при вопросах об устройстве кита — **не перечитывая исходники**.
> Правила (как работать) → `CLAUDE.md`. Состояние и история → `ЖУРНАЛ.html`.

## Философия
**Механизм > намерение.** Правила в тексте «выцветают» в длинной сессии — хук нет. Поэтому дисциплина вынесена из «Claude должен помнить» в «среда не даёт нарушить». Claude — сменный движок-исполнитель; система (правила + хуки + файлы) — носитель памяти и дисциплины между сессиями.

## Файлы
| Файл | Роль |
|---|---|
| `CLAUDE.md` | Правила (как работать). Лимит ~80 строк. |
| `ПРОЕКТ.md` | Видение / архитектура конкретного проекта. |
| `ЖУРНАЛ.html` | Живое состояние + история решений (WHY) + блок 📍ПЕРЕДАЧА (мини-ТЗ след. сессии). |
| `TODO.md` | Всё «вперёд»: Сейчас / Дальше / Заблокировано / 🧊 Идеи-когда-нибудь. Очередь — с «готово =»; Идеи — без. |
| `ХУКИ И СКРИПТЫ - АРХИТЕКТУРА.md` | Этот файл — карта системы работы (хуки/команды/скрипты). |
| `scripts/` | Утилиты и самотесты хуков. |
| `.claude/` | `settings.json` + `hooks/` + `commands/`. Защищённая зона — правки всегда спрашивают (by design). |

## Хуки (механические гарды)

**PreToolUse** — до вызова инструмента; `exit 2` = блок + сообщение Claude:
| Хук | Событие | Триггер → действие |
|---|---|---|
| `guard-bash.js` | Bash | опасные команды (`rm -rf`, `git push --force`, `reset --hard`, `curl\|sh`, запись в `.env`, `--no-verify`) **+ compound** (`&&`/`;`/shell-функции — механизация Правила 15) → блок (exit 2 перехватывает встроенный промпт). |
| `guard-files.js` | Edit/Write | защищённые пути (список PROTECTED) → блок. |
| `guard-subagent.js` | Agent/Task | задача субагенту без дисциплины shell → блок с напоминанием (Glob/Read/diff, без `&&`). Механизирует Правило 14. |

**Stop** — при попытке завершить ход; `exit 2` = не дать завершить:
| Хук | Триггер → действие |
|---|---|
| `visual-check.js` | правка вёрстки (css/jsx/html/php со стиль-маркерами) или живой страницы (Elementor/WPCode/POST) без скриншота после → требует скрин. Whitelist: ЖУРНАЛ/ПРОЕКТ/TODO/CLAUDE + `.claude/`. |
| `capitulation-check.js` | заявил «не могу / невозможно / проверь сам» без попытки решить (поиск или субагент в этом ходу) → требует сначала искать. Учитывает `Agent/Task` как поиск; НЕ флагает 👉 и слово «ограничение». |
| `todo-drain.js` | в «## Сейчас» TODO остались `[ ]` → гонит взять следующую (макс 5 авто/сообщение; маркер ❓ выпускает). |
| `journal-check.js` | была работа по коду, но нет записи в ЖУРНАЛ → требует зафиксировать WHY. TODO жёстко НЕ требует (его держит todo-drain). |

**UserPromptSubmit** — на сообщение Заказчика; stdout → контекст Claude:
| Хук | Триггер → действие |
|---|---|
| `remind-rules.js` | каждое N=15-е сообщение → впрыскивает весь CLAUDE.md (борьба с «выцветанием» правил). |
| `think-deeper.js` | кодовое слово «подумай / обдумай / а если подумать» → впрыскивает Second Thought Gate + велит начать ответ маркером «🧠 Deep Think активирован». |

## Команды (`.claude/commands/`, вызов `/имя`)
`/deploy` — цикл деплоя с проверкой до/после · `/journal` — обновить ЖУРНАЛ+TODO, перезаписать 📍ПЕРЕДАЧА · `/next` — взять задачу из TODO · `/plan` — план крупной задачи + самокритика + Second Thought Gate · `/rules` — перечитать правила · `/seccheck` — аудит безопасности перед релизом.

## Скрипты (`scripts/`)
- `test-*.sh` (7 шт: visual-check · capitulation · subagent · think-deeper · journal-check · guard-bash · guard-files) — самотесты хуков.
- `verify-todo-parse.py` — проверка, что TODO остаётся парсимым для `todo-drain` после украшения.
- `build-kit.py` — сборка чистого стартового кита в zip.

## Движок мышления (3 уровня)
1. **`/plan` Gate** (Правило 17) — для запланированных крупных решений.
2. **`think-deeper`** — углублённый отбор по слову «подумай», инлайн.
3. **anti-thrash** — после выбора решение = baseline; не переписывать рабочее без чёткого выигрыша.

Артефакт гейта: **Decision · 2 альтернативы · слабейшее допущение · Keep/Refine/Replace**. Цель — победить первую мысль, а не защитить её. Тривиальное (багфиксы, копи, CSS) — мимо гейта.

## Известные пределы (НЕ переисследовать)
- Permission-промпты «expansion obfuscation» и правки `.claude/` — **встроенные** гарды Claude Code, конфигом НЕ отключаются (bypassPermissions их не снимает). Лечение: shell сложнее одной команды → файл в `scripts/`.
- Stop-хуки эвристичны (regex) → возможны редкие ложные срабатывания. Тюнятся узкими whitelist'ами, не расширением.
- Точный %-контекста модель НЕ видит (только Заказчик на плашке) → Claude сигналит по ощущению, Заказчик решает (модель/новая сессия).
- Хуки требуют Node.js.

## Как тестировать
`bash scripts/test-<hook>.sh` — у каждого хука есть самотест. После правки хука — прогнать его тест (Правило 3).
