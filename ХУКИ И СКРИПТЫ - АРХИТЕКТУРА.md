# 🧭 ХУКИ И СКРИПТЫ — АРХИТЕКТУРА системы работы

> Что делает каждый хук / команда / скрипт и ПОЧЕМУ. Читать при вопросах об устройстве кита — **не перечитывая исходники**.
> Правила (как работать) → `CLAUDE.md`. Состояние и история → `ЖУРНАЛ.html`.

## Философия
**Механизм > намерение.** Правила в тексте «выцветают» в длинной сессии — хук нет. Поэтому дисциплина вынесена из «Claude должен помнить» в «среда не даёт нарушить». Claude — сменный движок-исполнитель; система (правила + хуки + файлы) — носитель памяти и дисциплины между сессиями.

## 🔧 Провижининг среды (Claude делает САМ — runbook для холодного старта)
Claude с обнулённой памятью должен поднять это без «а как подключить?» — кроме действий, физически требующих Заказчика (рестарт, UI-тумблеры). Это и есть «kit умеет привести среду в рабочее состояние».

### Playwright MCP — мобильные скриншоты (Правило 2)
UI-коннектор Playwright НЕ предлагает (он для remote HTTP/SSE; Playwright — локальный stdio). Ставится файлом. Триггер: нужен мобильный скрин, либо Заказчик просит.
1. **Проверь, подключён ли:** `ToolSearch "playwright browser screenshot"`. Есть `mcp__playwright__*` → уже стоит, используй. Пусто → ставь (ниже).
2. **Поставь САМ (Claude, Edit-ом — НЕ проси Заказчика):** в `~/.claude.json` (Windows: `C:\Users\<имя>\.claude.json`) добавь top-level ключ `mcpServers` (нет — создай первым ключом; есть — domerge `playwright` внутрь):
   ```json
   "mcpServers": { "playwright": { "command": "npx", "args": ["-y", "@playwright/mcp@latest"] } }
   ```
   ⚠️ Ключ ДОЛЖЕН быть top-level (рядом с другими корневыми), НЕ внутри `projects.<path>` (там — локальный для одного проекта). После правки проверь валидность: `node -e "JSON.parse(require('fs').readFileSync(process.argv[1]))" "<path>"` (тихо = ок).
3. **👉 Заказчик: рестарт Desktop** — единственное его действие (в живой сессии MCP не подхватывается). Зафиксируй handoff в ПЕРЕДАЧЕ, т.к. рестарт = конец сессии.
4. **После рестарта проверь сам:** ToolSearch playwright → загрузи `browser_resize`/`browser_navigate`/`browser_take_screenshot` → resize 390×844 → navigate URL → screenshot → Read PNG (реальный reflow в одну колонку = работает). Первый запуск тянет Chromium (~минута). Прод за Cloudflare → дай секунду на челлендж.
- user-scope (top-level) = работает во ВСЕХ проектах машины, ставится один раз. Per-machine (на новой машине — повторить).

### Bypass permissions — чтобы не было промптов (только Заказчик, UI)
1. Settings → Claude Code → включить тумблер **«Allow bypass permissions mode»** (разблокирует опцию).
2. Селектор режима у строки ввода (`Ctrl+Shift+M` → `5` = «Bypass permissions»). `settings.json defaultMode` десктоп ИГНОРИРУЕТ — рычаг только тут, **заново каждую сессию**. Claude напоминает на старте (чек-лист, шаг 6).

### Node.js v20+
Нужен хукам и Playwright. Проверка: `node -v`. Нет → 👉 Заказчик ставит (`nodejs.org` или `winget install OpenJS.NodeJS`).

## Команды (`.claude/commands/`, вызов `/имя`)
`/deploy` — цикл деплоя с проверкой до/после · `/journal` — обновить ЖУРНАЛ+TODO, перезаписать 📍ПЕРЕДАЧА · `/next` — взять задачу из TODO · `/plan` — план крупной задачи + самокритика + Second Thought Gate · `/rules` — перечитать правила · `/seccheck` — аудит безопасности перед релизом.

## Скрипты (`scripts/`)
- `test-*.sh` (9 шт: visual-check · capitulation · subagent · think-deeper · journal-check · guard-bash · guard-files · anti-push · guard-askquestion) — самотесты хуков.
- `rebuild-kit.ps1` — пересборка эталон-архива `claude-kit-starter.zip` из generic-файлов проекта (PowerShell, сохраняет UTF-8 имена).

## Движок мышления (3 уровня)
1. **`/plan` Gate** (Правило 17) — для запланированных крупных решений.
2. **`think-deeper`** — углублённый отбор по слову «подумай», инлайн.
3. **anti-thrash** — после выбора решение = baseline; не переписывать рабочее без чёткого выигрыша.

Артефакт гейта: **Decision · 2 альтернативы · слабейшее допущение · Keep/Refine/Replace**. Цель — победить первую мысль, а не защитить её. Тривиальное (багфиксы, копи, CSS) — мимо гейта.

## Известные пределы (НЕ переисследовать)
- Правки `.claude/` — **встроенный** гард Claude Code, спрашивают ВСЕГДА (даже в bypass, by design). А промпт «expansion obfuscation» на compound-команды **снимается Bypass-режимом** (проверено 2026-06-15) — поэтому блок compound из `guard-bash` убран. Bypass на Desktop = селектор у строки ввода (`Ctrl+Shift+M`→5), НЕ `settings.json defaultMode`.
- Stop-хуки эвристичны (regex) → возможны редкие ложные срабатывания. Тюнятся узкими whitelist'ами, не расширением.
- Хуки требуют Node.js.

### 🔄 Раскатка обновления в проект (по фразе Заказчика)
Заказчик кидает свежий `claude-kit-starter.zip` в проект и говорит фразу → выполни:
1. **Своя версия** — штамп в шапке `CLAUDE.md` (`kit @ дата`).
2. **Распакуй** свежий архив (лежит в проекте) — на Windows **через PowerShell `Expand-Archive`, НЕ `unzip`** (unzip корёжит кириллические имена файлов).
3. **Запусти `apply-kit` — НЕ копируй руками** (детерминированный синк + сверка полноты; убирает «думал, что внедрил, а нет»). Скрипт лежит В АРХИВЕ (`scripts/apply-kit.ps1`) и в проекте после первой раскатки. Из распакованного архива нацель на проект:
   `pwsh -NoProfile -ExecutionPolicy Bypass -File <распак>\claude-kit-starter\scripts\apply-kit.ps1 -Project <корень проекта> -Zip <архив>` (нет pwsh → `powershell` вместо `pwsh`; `-ExecutionPolicy Bypass` обязателен для 5.1; файлы в UTF-8+BOM, кириллица не ломается).
   Он копирует generic-файлы по различиям, **НЕ трогает** `ПРОЕКТ.md` / `ЖУРНАЛ.html` / `TODO.md` / `ПЕРЕДАЧА.md` / `guard-files.js`, и СВЕРЯЕТ до/после: вывод КОРОТКИЙ — сводка + заметный баннер-вердикт последней строкой («✅ РАСХОЖДЕНИЙ НЕТ» либо «⚠️ РАСХОЖДЕНИЯ: список», exit 1). `-DryRun` — показать без копирования; `-Details` — полные списки.
4. **Сверь «История версий кита»**: записи новее твоего штампа = что изменилось (для отчёта/понимания; копирование уже сделал apply-kit).
5. **Бампни штамп** в шапке CLAUDE.md — обычно его уже принёс apply-kit (CLAUDE.md generic); если нет — проставь версию архива.
6. **Прогони самотесты:** хуков (`rm -rf /несуществующая-тест` → должно блокнуть) + `scripts/test-*.sh` + `pwsh -File scripts/test-apply-kit.ps1`.
7. **Отчитайся КОРОТКО — вердиктом, не простынёй.** Финал отчёта = заметная строка `✅ Раскатка <версия> — РАСХОЖДЕНИЙ НЕТ` (или `⚠️ РАСХОЖДЕНИЯ: <список>`). Максимум +1 строка (самотесты зелёные?). НЕ пересказывай весь вывод apply-kit и changelog — Заказчику нужен вердикт, а не отчёт.

**Универсальная фраза Заказчика** (одна на ВСЕ раскатки — и первую, и последующие; самодостаточна, не зависит от того, есть ли уже рунбук в проекте):
> «Обнови стартер-кит. Распакуй свежий архив через PowerShell `Expand-Archive` (не `unzip` — корёжит кириллицу) и запусти из него `pwsh -NoProfile -ExecutionPolicy Bypass -File scripts/apply-kit.ps1 -Project <корень проекта> -Zip <архив>` (нет pwsh → `powershell`): он синкнет generic-файлы со сверкой полноты, проектные (ПРОЕКТ / ЖУРНАЛ / TODO / ПЕРЕДАЧА / guard-files.js) не тронет. Прогони самотесты. Отчитайся КОРОТКО — одной заметной строкой-вердиктом apply-kit (✅ расхождений нет / ⚠️ список), без пересказа.»

Фраза **самодостаточна** — годится и для первой раскатки в проект, и для всех последующих (не зависит от того, есть ли уже рунбук внутри). Запоминать две фразы не нужно.
